Sorensen Prototypes & MVP Case Studies

The following are a few highlights from my experience developing prototypes and practicing lean, MVP principles to maximize learning and minimize development time.

Simplify a complex app for faster testing.

Recog, 2017

Try ReCog Demo
Challenge:
The value of a smart note visualizer can take weeks of accumulated notes to realize. Find a way to demonstrate and test the value of the app in a few minutes to speed up learning and iteration.
Role:
One man team, including product spec, design, coding, deployment, and user testing.
Approach:
  • Organize demo in a wizard format to streamline the process of structuring notes, generating sample notes, and creating visualizations.
  • Allow users the choice to personalize or to skip steps in favor random / pregenerated data.
  • Make inferences about which visualizations to present based on the type of data in the notes.
Outcome:
Can test and collect feedback in 10-15 minutes or less. (Testing still in progress).

Building a Culture of Validation and Lean Development

Education Elements, 2014-2016

Challenge:
Education Elements released one large product with lots of features in the fall, often with little user testing or validation. Products were underused and discarded at the end of the cycle.
Team Members:
Product manager, visual designer, front-end and back-end teams with several members each.
Role:
Team lead for the front-end.
Approach:
  • Change the product development process to include weekly user testing of in-development projects.
  • Organize product development cycle around a strict two-month deadline, rather than extending development to include as many features as possible.
  • Enable faster development agility with better tooling.
Personal Contributions:
  • Inteview UX designers to develop user testing processes.
  • Run user testing sessions.
  • Advocate adoption of 2-month MVP cycle, help with MVP scoping and planning.
  • Develop a flexible build system that enabled rapid development and release of new products.
  • Design and implement a living style guide.
Outcome:
Regular user testing of the product pre-release, 2-month MVP delivered in 6 weeks, building tools enabled a small but crucial and time-sensitive new app to be developed, tested, and release in 90 minutes.

Test for Value in New Application of Existing Data

LinkedIn, 2014

Current version of recommended courses on LinkedIn
Challenge:
LinkedIn had an opportunity to leverage user profile data to recommend online courses and provide greater professional value to users.
Team Members:
Product manager, Visual Designer, Data Scientist/API Programmer, Developer / Release Engineer.
Role:
Developer / Release engineer.
Approach:
  • Use skills and other proprietary data to match users to relevant online courses.
  • Display courses with an interface for saving, sharing, and marketing courses as complete.
  • Release in company wide internal test.
Personal Contributions:
  • Consume API in Play, a Java-Based MVC.
  • Use LinkedIn's live style guide and custom code to implement front-end interface.
  • Configure and deploy the new app.
Outcome:
Course Match prototype patented and released company-wide to favorable feedback. A version of the feature is now live.
